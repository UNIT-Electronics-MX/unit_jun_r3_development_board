UNIT JUN R3 Development Board

Introduction

UNIT JUN R3 is a versatile and modular development board based on the ATmega328P microcontroller, compatible with the UNO-style form factor. Designed for rapid prototyping, it is well-suited for embedded systems education, interactive projects, and wearable technology. The board offers flexible power input options, modern connectivity, and user-friendly interfaces to streamline development workflows. It also features an integrated 5√ó5 NeoPixel LED matrix, ideal for creating visual indicators, feedback systems, or simple dynamic displays.

üß™ Use Cases

  • Prototyping: Rapidly build and evaluate embedded system concepts.
  • Education: Ideal for teaching microcontroller fundamentals and electronics.
  • Wearables: Compact form factor and flexible power options for wearable tech.
  • Interactive Displays: Leverage the integrated NeoPixel matrix for dynamic visual feedback.

Getting Started

Quick Start Guide

1. Hardware Setup

  1. Connect to Computer: Use the USB-C cable to connect your UNIT JUN R3 to your computer
  2. Install Drivers: Most operating systems will automatically recognize the board
  3. Verify Connection: Check that the power LED is on

2. Development Environment

Arduino IDE

  1. Install Arduino IDE
  2. Add the ATmega328P board support if needed
  3. Select Arduino Uno as the board type
  4. Install required libraries:
    • Adafruit NeoPixel Library

Platform.io

[env:uno]
platform = atmelavr
board = uno
framework = arduino
lib_deps = 
    adafruit/Adafruit NeoPixel@^1.10.5

3. First Program

Try the basic NeoPixel example to verify everything works:

#include <Adafruit_NeoPixel.h>

#define PIN        8
#define NUMPIXELS 25

Adafruit_NeoPixel pixels(NUMPIXELS, PIN, NEO_GRB + NEO_KHZ800);

void setup() {
  pixels.begin();
  pixels.setBrightness(50);
}

void loop() {
  pixels.clear();
  pixels.setPixelColor(0, pixels.Color(255, 0, 0)); // Red
  pixels.show();
  delay(1000);
}

4. Next Steps

  • Explore the Examples section
  • Check out QWIIC sensor integration
  • Try different NeoPixel patterns
